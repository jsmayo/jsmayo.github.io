Shopping Checkout Simulator

Details of the project are displayed below. Please click the images for a larger view.

Final UML Design

I was given the task of designing a shopping simulation program to generate conditions that would model both the average wait and shopping time of a customer in this specific client's store. Additionally, conditions for specific cart type and register classifications, such as 'Express', 'Regular', and 'Special Handling', were accounted for by manipulating parameters that modeled real world data from the client.

Basic User Interface

The user interface of the program allowed varying the animation speed, total number of shopping carts, and total number of checkout registers. Values for these parameters are simulated when the user clicks 'Start' and are displayed to the right of those variables after the simulation terminates. A progress bar, as well as, basic animation of the simulation are available for instant user feedback.

Process Time Calculation

Calculation of the checkout time begins when the user enters the simulated register's checkout line. If there is no wait, then the process time is the total time required to complete the checkout. If there is a wait, then process time is the wait time of the customers infront of the current cart, plus the time that will be required for that cart to checkout.
